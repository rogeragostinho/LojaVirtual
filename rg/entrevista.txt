1)R: Uma base de dados relacional organiza os dados em tabelas relacionadas entre si através de chaves primárias e estrangeiras. É importante porque evita redundância, mantém a integridade dos dados e facilita consultas complexas.

2)R: Normalização é o processo de organizar os dados para reduzir redundâncias e evitar inconsistências.

3)R: Uma transação é um conjunto de operações executadas como uma única unidade. Ou todas são concluídas com sucesso ou nenhuma é aplicada.

4)R: Autenticação responde "Quem és?" e autorização responde "O que podes fazer?".

5)R: Eu procuraria implementar autenticação através de tokens, validar todos os dados recebidos, usar HTTPS, controlar permissões dos utilizadores, proteger contra SQL Injection utilizando ORM ou queries parametrizadas, e nunca expor informações sensíveis nas respostas da API.

6)R: Versionamento permite acompanhar alterações no código ao longo do tempo. Utilizo Git para registar mudanças, criar branches para novas funcionalidades e fazer merge quando as alterações estão concluídas.

7)R: Clean Code
É escrever código fácil de ler, compreender e manter.
Exemplos:

Nomes claros.
Funções pequenas.
Evitar duplicação.
Comentários apenas quando necessários.

SOLID
Memoriza isto:

S — Single Responsibility Principle
Uma classe deve ter apenas uma responsabilidade.

O — Open/Closed Principle
O código deve permitir extensão sem alterar o que já funciona.

L — Liskov Substitution Principle
Classes filhas devem poder substituir as classes pais sem quebrar o sistema.

I — Interface Segregation Principle
Interfaces devem ser específicas e não obrigar implementação desnecessária.

D — Dependency Inversion Principle
Devemos depender de abstrações e não de implementações concretas.

2. PERGUNTAS DE BACKEND QUE PODEM APARECER
O que acontece quando digitamos um site no navegador?
Resposta resumida:
O navegador procura o IP através do DNS, envia uma requisição HTTP ao servidor, o backend processa a requisição, consulta a base de dados se necessário e devolve uma resposta ao navegador.

O que é uma API?
Uma API é uma interface que permite a comunicação entre diferentes sistemas através de requisições e respostas.

Diferença entre GET e POST?

GET:
Consultar dados.

POST:
Criar dados.

PUT e PATCH?
PUT:
Atualização completa.

PATCH:
Atualização parcial.

O que é JSON?
É um formato de troca de dados muito utilizado entre frontend e backend.

O que é uma chave estrangeira?
Campo que referencia uma tabela diferente.

O que é um JOIN?
Permite combinar dados de várias tabelas relacionadas.

O que é um índice?
Estrutura que acelera pesquisas na base de dados.

Quando usar índice?

Campos pesquisados frequentemente:
email, número da apólice, NIF
